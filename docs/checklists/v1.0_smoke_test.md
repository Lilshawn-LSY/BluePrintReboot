# BluePrintReboot v1.0 Smoke Test Checklist

> Pre-release checklist for a disposable or backed-up workspace. This does not publish v1.0.

## 1. Fresh Clone and Runtime

- [ ] Clone the repository into a new directory.
- [ ] Confirm `data/`, `papers/`, `notes/`, and `exports/` exist with their `.gitkeep` placeholders.
- [ ] Create and activate a virtual environment.
- [ ] Install `requirements.txt` without errors.
- [ ] Optionally install `requirements-optional.txt` and confirm MarkItDown availability.
- [ ] Run `python scripts/smoke_check.py`; confirm there are no failed checks.
- [ ] Run `streamlit run app.py`; confirm the app opens without an exception.
- [ ] Confirm an empty workspace shows helpful empty-library messages rather than errors.

## 2. Scan and Library Navigation

- [ ] Copy one or two non-sensitive sample PDFs into `papers/`.
- [ ] Select **Scan papers**.
- [ ] Confirm each PDF appears once in Library and its managed path is under `papers/`.
- [ ] Run a second scan and confirm records are not duplicated.
- [ ] Search and filter the Library list.
- [ ] Open a paper and confirm Paper Detail loads.

## 3. Metadata and Crossref

- [ ] Edit title, authors, year, status, priority, and tags; save and reload.
- [ ] Enter or extract a DOI and confirm normalization remains stable.
- [ ] While online, run **Settings > External Services > Crossref Diagnostics**.
- [ ] Request a Crossref preview and confirm metadata is not applied until explicitly accepted.
- [ ] Confirm empty Crossref fields do not erase existing metadata.
- [ ] While offline or proxy-blocked, confirm diagnostics show a concise classified error and local workflows remain usable.

## 4. Reader Workspace and Notes

- [ ] Open the PDF with the default stable viewer.
- [ ] Edit and save the Markdown note, reload the page, and confirm the text persists.
- [ ] Create a structured note block for at least two block types.
- [ ] Edit one block and confirm its identity remains stable.
- [ ] Append a block snapshot to Markdown and confirm later block edits do not rewrite the snapshot.
- [ ] Delete a disposable block through its confirmation flow.

## 5. Tags and Projects

- [ ] Preview tag suggestions and apply selected suggestions.
- [ ] Review **Settings > Library Maintenance > Tag Rules**.
- [ ] Open Tag Manager and sanity-check canonical, alias, unknown, and unused tag views.
- [ ] Preview a tag merge without applying it; confirm no paper tags change.
- [ ] Create a disposable project.
- [ ] Link a paper and a structured note block to the project.
- [ ] Edit a link note/type, then unlink a disposable link through confirmation.

## 6. Paper Hygiene and Identity

- [ ] Record the selected paper's `paper_id` from `data/paper_index.csv`.
- [ ] Fill title, authors, and year metadata.
- [ ] Preview **Settings > Library Maintenance > Paper Hygiene**.
- [ ] Confirm missing year plus author blocks placeholder-style renaming.
- [ ] Apply one safe rename and confirm the PDF remains under `papers/`.
- [ ] Confirm `paper_id`, Markdown notes, note blocks, project links, and extracted-text cache identity remain unchanged.
- [ ] Run **Scan papers** again and confirm no duplicate paper record appears.

## 7. Full-Text Cache

- [ ] Extract full text and inspect status, backend, character count, and preview.
- [ ] Re-extract and confirm the workflow remains explicit.
- [ ] If practical, alter a disposable PDF and confirm stale-cache detection reports the changed hash.
- [ ] Confirm a failed recovery preserves a previous usable cache.
- [ ] Delete a disposable cache through confirmation.

## 8. Drive Inbox Import

- [ ] Configure a disposable local inbox outside `papers/`.
- [ ] Scan the inbox and preview one PDF candidate.
- [ ] Import it and confirm the source remains untouched.
- [ ] Confirm the copied file is under `papers/` and the indexed filepath never points to the inbox.
- [ ] Confirm filename collisions and duplicate content are blocked without overwrite.
- [ ] Confirm configuring `papers/` itself as the inbox is rejected.

## 9. Library Health Check

- [ ] Run **Settings > Library Maintenance > Library Health Check** on a healthy sample workspace.
- [ ] Temporarily create a missing-PDF condition and confirm it is reported.
- [ ] Temporarily add an unindexed PDF and confirm it is reported.
- [ ] Review duplicate DOI/filename, missing metadata, orphan, noncanonical path, and stale-cache sections.
- [ ] Restore the disposable test workspace to a healthy state.

## 10. Backup Snapshot and Manifest

- [ ] Create a light snapshot and confirm it excludes `papers/` PDFs.
- [ ] Create a full snapshot after acknowledging PDF inclusion and confirm it includes managed PDFs.
- [ ] Open each ZIP without extracting over the active library.
- [ ] Inspect `manifest.json` and verify the fields listed in the [new-PC restore checklist](new_pc_restore_checklist.md#manifest-expectations).
- [ ] Confirm `.git`, `.venv`, `__pycache__`, `.pytest_cache`, and previous exports are absent.
- [ ] Confirm snapshot creation does not modify library files.

## 11. Manual Restore Rehearsal

- [ ] Follow [new_pc_restore_checklist.md](new_pc_restore_checklist.md) using a separate temporary clone.
- [ ] Keep the app stopped during extraction.
- [ ] Run Scan papers and Library Health Check after restore.
- [ ] Compare index rows, PDF count, notes, note blocks, projects, and project links with the source workspace.
- [ ] Retain the original snapshot until verification is complete.

## 12. Final Automated Validation

- [ ] Run `python scripts/smoke_check.py` with zero failures.
- [ ] Run `python -m pytest` with all tests passing.
- [ ] Review the unpublished `docs/release_notes/v1.0_draft.md`.
- [ ] Confirm known limitations are still accurate.
- [ ] Publish no GitHub Release until explicitly approved.
