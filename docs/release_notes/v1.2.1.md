# v1.2.1-full-stack-validation-gate

## Summary

This release closes the validation gap between the existing Streamlit application, read-only FastAPI layer, and v1.2.0 TypeScript frontend shell. It adds no product feature and changes no user-data, API-response, storage, note, snapshot, or restore contract.

## Changes

- Shared PowerShell Node resolver: explicit `-NodeHome`, then `BLUEPRINT_NODE_HOME`, then `PATH`; both `node.exe` and `npm.cmd` are required and Node below 22.13.0 is rejected.
- Deterministic `frontend_setup.ps1` using only `npm ci`, plus opt-in frontend setup from `dev_setup.ps1`.
- Default full `dev_check.ps1` gate for smoke, full pytest, frontend lint, and one frontend build/test pass. `-PythonOnly` and `-SmokeOnly` are explicitly partial and non-release-qualified.
- Optional ignored `artifacts/validation-summary.json` containing normalized phase states and runtime/Git metadata without command output, secrets, environment values, or private library paths.
- Focused same-origin bridge tests for exact GET allowlisting, query forwarding, upstream 404 preservation, generic upstream/network 503 behavior, and absence of write methods.
- Independent GitHub Actions jobs for Python 3.12 and the minimum supported Node 22.13.1 line.

## Compatibility and next milestone

The four API route shapes remain GET-only: `/health`, `/library/status`, `/papers`, and `/papers/{paper_id}`. Streamlit remains the write, note, PDF, and maintenance interface. The next product milestone is a separately designed read-only Reader/PDF vertical slice; PDF.js, PDF-serving routes, notes, and all write methods remain out of scope here.
