# v1.2.1-full-stack-validation-gate

## Summary

v1.2.1 joined the established Streamlit application, read-only FastAPI layer, and v1.2.0 TypeScript shell under one reproducible local gate. It added portable Node resolution, deterministic frontend setup, bridge behavior tests, optional machine-readable evidence, and separate Python/frontend workflow jobs. It added no product feature and changed no user-data or API contract.

## Historical address-family observation

The v1.2.1 runtime report recorded Vinext listening on IPv6 loopback `::1`, with localhost reachable and the documented IPv4 frontend address unreachable. That discrepancy did not invalidate the automated build/test evidence, but it left the exact frontend launch contract unresolved. v1.2.2 later traced the discrepancy to an unsupported Vinext `--host` argument and corrected the launcher to use `--hostname 127.0.0.1`.

## Compatibility and remaining v1.2.1 gaps

The API remained GET-only at `/health`, `/library/status`, `/papers`, and `/papers/{paper_id}`. Streamlit remained the write, note, PDF, metadata, and maintenance interface. The exact outstanding historical items were Library rendering, Paper Detail rendering, clean-PC restore rehearsal, and GitHub-hosted CI evidence.
