# BluePrintReboot v1.0.26 Streamlit Finalization and API Contract Freeze

## Root cause and fix

Reader toolbar manual and selected-suggestion tag actions wrote directly to `paper_index.csv` and bypassed the separate UI-only Reading Note header refresh helper. Other metadata paths invoked that helper, producing inconsistent behavior. v1.0.26 replaces those paths with one typed, Streamlit-independent metadata mutation coordinator used by Reader, Edit metadata, DOI, DOI-less, selected tags, and Crossref acceptance.

The coordinator normalizes and compares changes, skips no-ops, atomically saves the index, reloads the paper, refreshes clean note headers, queues dirty-draft header convergence without writing the body, and reports index failure or post-index note partial failure. Explicit Save always converges the latest canonical header with the latest draft body.

Reader Save uses a two-run Streamlit transition: the widget-rendering pass persists and queues a paper-scoped saved result without mutating the instantiated widget key; the rerun consumes saved text, baseline, clean state, cleared header-refresh state, and success feedback before recreating the text area. A Streamlit AppTest covers this lifecycle.

## Boundary freeze

- Adds JSON-safe `HealthSummary`, `LibraryStatus`, `PaperListItem`, `PaperDetail`, and `ReaderSnapshot` builders with deterministic defaults and safe relative paths.
- Adds a non-mutating index snapshot reader; contract construction never migrates or saves the index.
- Adds a read-only disposable restore readiness helper that verifies the original snapshot and rejects missing, non-empty, active-repository, or unsafe targets. It never extracts.
- Updates Reader/lifecycle parity contracts for future adapters. No API route or frontend code is included.

## Manual evidence

The user reported the v1.0.24 Reader and v1.0.25 lifecycle manual validations completed before this release request. The tag/header divergence was discovered afterward during actual use. The user subsequently reported v1.0.26 Sections A through H passed: the prior `StreamlitAPIException` is fixed, Save convergence passed, and paper-navigation discard was explicitly accepted. G4 is closed after contract alignment.

Accepted navigation behavior: dirty drafts survive same-paper metadata reruns but are never autosaved. Switching papers, browser refresh, or application restart discards unsaved content. Returning to a paper loads its last explicitly saved Reading Note; no per-paper persistent draft store is required.

The real clean-PC restore rehearsal was not performed in this environment and remains an explicit user check.
