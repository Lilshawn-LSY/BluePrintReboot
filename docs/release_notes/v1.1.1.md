# v1.1.1-paper-api

## Summary

This release adds the first stable read-only Paper API on top of the v1.1.0 FastAPI foundation. Streamlit remains the primary interface and its storage and mutation workflows are unchanged.

## API contract

- `GET /papers` returns a `PaginatedPaperList` envelope.
- `limit` defaults to 20 and accepts 1 through 100.
- `offset` defaults to 0 and must be non-negative.
- `archive_status` defaults to `active` and accepts exactly `active`, `archived`, or `all`.
- `total` is the matching count before pagination; `has_more` indicates whether another matching item follows the returned page.
- Results are ordered by case-insensitive title, then stable `paper_id`.
- `GET /papers/{paper_id}` returns active or archived `PaperDetail`; unknown IDs return `{"detail": "Paper not found."}` with HTTP 404.

## Architecture and safety

The API depends on `build_paper_list_items()` and `build_paper_detail()` and maps their frozen JSON-safe contracts into strict Pydantic models. It does not load CSV rows, parse storage data, expose absolute paths or hashes, or import Streamlit. Malformed public identity/path data fails safely instead of becoming a misleading 404.

Archive state reuses the existing `is_archived` lifecycle field. Missing archive values remain active, and reading progress continues to use the separate `status` field.

## Deferred

Journal, abstract, keywords, and arXiv fields require a future explicit domain-contract extension and are not read directly from the index by the API. Search, custom sorting, Reader/PDF/note/tag/project endpoints, write/archive actions, authentication, and frontend work remain excluded.
