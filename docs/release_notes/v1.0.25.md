# BluePrintReboot v1.0.25 Lifecycle and Recovery Closure

## Behavior Changes

- Health Check emits structured, storage-class-aware corrupt-state records for app-owned JSON, extracted-text cache pairs/text/metadata, and PaperTextProfile caches.
- Every diagnosed app-owned file can be exported byte-for-byte; rebuildable caches can be explicitly quarantined and restored with verification and conflict protection.
- Exact unindexed duplicate ignores persist atomically and apply only to the same workspace-relative path and SHA-256; Unignore is available.
- Archive is reversible metadata visibility, separate from reading status, and never moves or deletes a file.
- Pure summaries expose stable active/archived, missing-PDF, duplicate, ignored, corruption, quarantine, and degraded-library semantics without adding an API.

## Accepted Limitations

- Duplicate merge and duplicate-file deletion remain deferred.
- Critical user JSON is never automatically repaired or reset.
- Cache rebuild remains a separate explicit action after quarantine.
- Tag-governance expansion remains frozen.
- FastAPI, API routes, frontend frameworks, PDF.js, databases, OCR, ML, and LLM tagging remain out of scope.
