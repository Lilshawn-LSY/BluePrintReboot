# v1.1.0-fastapi-readonly-foundation

## Summary

This release adds the initial local FastAPI foundation while preserving the existing Streamlit application as the primary interface. The API is read-only and adapts the frozen v1.0.26 domain builders without duplicating storage parsing or health logic.

## Included

- Dedicated `api` package with an application factory and `api.main:app` entrypoint.
- `GET /health`, returning the exact `HealthSummary` contract.
- `GET /library/status`, returning the exact `LibraryStatus` contract.
- Strict Pydantic response models that forbid undeclared fields.
- Stable generic HTTP 503 handling for unavailable read models.
- A local-only PowerShell launcher bound to `127.0.0.1`.
- Smoke validation for importability, version metadata, route scope, and GET-only behavior.
- Focused synthetic/override-based API tests that do not use the user's library.

## Compatibility

- Streamlit remains fully supported and is still the primary UI.
- Existing read-model builders and their JSON-safe contracts are unchanged.
- No user-data migration or automatic data modification is introduced.

## Explicitly deferred

Paper list/detail routes, Reader routes, PDF serving, note/tag/project endpoints, all write APIs, CORS, authentication, background tasks, caching, databases/ORMs, frontend frameworks, cloud sync, upload/import/delete/archive/restore/repair actions, and `paper_id` redesign are not included.

## Start the API

Run `.\scripts\run_api.ps1`, then open `http://127.0.0.1:8000/docs`. The health endpoints are available at `/health` and `/library/status`.
