# v1.2.0-frontend-application-shell

## Summary

This release introduces a professional desktop-first web application shell for BluePrintReboot while preserving Streamlit as the primary operational interface. The visual system is calm, neutral, compact, and designed for long research sessions rather than marketing or analytics presentation.

## Routes and shell

The persistent shell contains a left sidebar, compact route-aware header, and efficient content region across `/`, `/dashboard`, `/library`, `/papers`, `/papers/{paperId}`, `/projects`, `/tags`, and `/settings`. Navigation uses labeled icons, client-side links, active-route state, semantic landmarks, and visible keyboard focus.

## Real integrations

Dashboard, Library, Papers, and Paper Detail consume the existing health, library-status, paper collection, and paper-detail contracts. All access flows through one typed client and an allowlisted same-origin GET bridge to local FastAPI. Offline, empty, loading, validation/error, and unavailable states are explicit; no fake papers, projects, tags, counts, or activity are substituted.

Projects, Tags, and Settings are structured placeholders that state their purpose, missing API dependency, and current Streamlit location. They provide no pretend actions.

## Design system

Central CSS tokens define neutral surfaces, forest accent, semantic states, compact spacing, type scale, borders, radii, shadow, sidebar/header dimensions, content padding, focus ring, and z-index layers. Shared shell, navigation, headers, sections, toolbar, status badges, search shell, async states, table shell, and detail panel components keep route presentation consistent.

The layout supports desktop and laptop widths, horizontal table scrolling, a labeled compact navigation grid at narrow widths, reduced motion, and semantic focus/alert/status behavior.

## Compatibility and limits

The Streamlit application and Python storage/domain layers are unchanged except for the shared application version. Write actions, note editing, PDF rendering, project/tag mutation, advanced search, authentication, synchronization, and mobile polish remain deferred.

The default frontend bridge targets `127.0.0.1:8000`. A remotely hosted shell cannot reach a user's localhost API; useful hosted data integration requires a separately reachable compatible API and explicit configuration.
