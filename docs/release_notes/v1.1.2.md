# v1.1.2-rich-paper-metadata

## Summary

This release extends the stable read-only `PaperDetail` domain and API contracts with rich citation metadata while preserving the v1.1.1 collection, pagination, archive, error, and path-safety behavior.

## Domain and API contract

`PaperDetail` adds:

- `authors: list[str]`
- `journal: str`
- `abstract: str`
- `keywords: list[str]`
- `arxiv_id: str`

`first_author` remains available for compatibility. `PaperListItem` does not add these fields, keeping collection responses compact.

## Sources and normalization

Authors, journal, abstract, and keywords come only from their canonical paper-index columns. Authors use the repository's semicolon serialization, preserving order and commas inside names. Keywords use the canonical comma serialization and preserve first-seen order. Abstracts are returned completely after trimming outer whitespace. Missing, None, and NaN values become empty strings or lists.

arXiv identity follows the established Reading Note rule: a normalized explicit value wins when available; otherwise the first identifier detected from stored DOI, filename, title, abstract, and keywords is returned. There is no profile-cache fallback, network lookup, automatic enrichment, PDF parsing, or extracted-text parsing during reads.

## Compatibility and safety

Active, archived, and missing-PDF details remain readable. Unknown IDs remain 404 and genuine read failures remain generic 503 responses. The adapter continues explicit field allowlisting and rejects absolute/traversal PDF paths. Existing indexes without optional metadata columns remain readable without write-time migration.

## Deferred

Search, custom sorting, metadata writes, network enrichment, full-text/PDF/Reader routes, frontend work, authentication, and database changes remain excluded.
