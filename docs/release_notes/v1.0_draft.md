# BluePrintReboot v1.0 Release Notes — Draft

> Draft only. This file is not a published GitHub Release and does not represent a final release commitment.

## Direction

BluePrintReboot is a local-first research paper library for readers who want their PDFs, metadata, notes, and research organization to remain understandable and portable on their own machine. The v1.0 direction keeps the Streamlit interface and stable `paper_id` identity model while bringing the current workflows into a coherent daily-use experience.

## Current Highlights

- **Local paper library** — `papers/` remains the canonical managed PDF location, with metadata in `data/paper_index.csv`.
- **Metadata and DOI workflow** — local DOI extraction, manual metadata editing, and optional Crossref preview/acceptance with polite access and classified diagnostics.
- **Full-text extraction cache** — user-triggered extraction, backend diagnostics, SHA-256 freshness checks, and safe stale-cache recovery.
- **Reader Workspace** — PDF reading, Markdown notes, reading status, priority, and tag controls in one workspace.
- **Structured note blocks** — summaries, claims, methods, evidence, questions, ideas, and limitations stored by stable paper identity.
- **Project linking** — papers and note blocks can be linked into lightweight research projects with relationship types and notes.
- **Tag governance** — deterministic suggestions, canonical tags, aliases, audits, and previewed merges.
- **Paper Hygiene** — previewed human-readable PDF filenames without changing `paper_id` or paper-linked data.
- **Drive Inbox Import** — explicit copy-only import from a Drive-synced local folder into the canonical library.
- **Backup Snapshot** — light and full timestamped archives with checksummed manifests.
- **Library Health Check** — read-only diagnostics for missing, duplicate, orphaned, incomplete, stale, and noncanonical library records.

## UX Cleanup Before v1.0

Version 0.9.8 reorganizes Settings into System, Library Maintenance, External Services, and Backup. It standardizes labels, separates passive diagnostics from maintenance actions, simplifies README startup guidance, and collects development commands away from the everyday user workflow.

## Safety and Portability

- Runtime library data remains outside Git.
- Existing paper IDs are never regenerated by maintenance workflows.
- Destructive actions require confirmation where implemented.
- Inbox import copies rather than moves source files and never overwrites a managed PDF.
- Snapshot creation is additive; restore remains a documented manual process.

## Known Gaps

The current v1.0 direction does not include OCR, visual highlighting, coordinate annotations, Google Drive API/OAuth, Zotero integration, relation graphs, or an automated destructive restore workflow.
